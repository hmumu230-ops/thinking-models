# 防伪通用化机制 v1 · 说明（默认特例 · 通用靠挣）

## 它要防的那一种过拟合
> 「一个模型刚好能解释某个现象，就被当成对这一类事普遍成立的通则。」

解释是无限弹性的——回头看，什么都能解释，所以「它解释得通」不是证据。而且你往往是**因为它能解释这个现象才挑中它的**，于是「它正好能解释」是被你的挑选保证的，循环论证（先开枪再画靶）。N=1 的解释力，信息量约等于零。

## 机制（直接用库里的模型推导出来）
- **逆向思维**：先列「怎样保证库一定被伪通用污染」（入库凭故事判通用 / 不记使用 / 不看被拒的 / 降级难 / 按产量奖励代理），再**反过来**就是规格。
- **贝叶斯**：通用性是一个**随每次独立使用而更新的后验**，不是入库时的标签。入库 = 低先验 = 默认「特例」。
- **一种证据**：你真实的、跨情境的使用 → 记进 `proven_domains`。≥3 个互相独立的域 → 够格升「通用」。
- **幸存者偏差**：保留 `拒绝档案.md`，每周反审有没有误杀。没有坟场就只见幸存者。
- **可证伪性**：机制的「失效信号」= 升过「通用」却被剪 / 从不跨域用。每周统计这个**错误率**；出现就把闸再收紧一格。
- **反脆弱 + 安全边际**：别追求判得准（不可能），要让判错**便宜可逆**——降级和升级一样容易；每次「误升被剪」都反过来收紧闸门，机制从自己的错误里变强。
- **奥卡姆 + 认知负荷**：四道闸坍缩为**一条贝叶斯规则**，避免机制自身过拟合、避免你那 2 分钟做不到。

## 字段约定（见 `思维模型库.js`）
- `scope_type`：`"特例"` → `"领域"` → `"通用"`。新入库**强制「特例」**；52 个种子模型 grandfather 视作「通用」。
- `proven_domains`：你真实用过它的「互相独立情境」清单（这是通用性唯一可信的证据）。
- `general`：仅「入库先验提示」，不等于已验证的通用性。

## 你唯一不可省的动作
> **每次真用到某个模型，就给它的 `proven_domains` 记一笔**（写下用在什么独立情境）。

没有这一步，机制空转——通用性永远无法被证明，特例永远升不了级。每周代理只能依据它来提议升级。

## 角色分工
| 环节 | 谁 | 防过拟合 |
|---|---|---|
| 找候选 / 初筛 / 摆证据 | 每日代理 | 缺口制导 + 贝叶斯准入 + 双向辩护 |
| 隔离（一律进待审、默认特例） | 系统 | 永不自动判通用 |
| **准入 / 升级 / 记使用** | **你** | 人是最终正则项 |
| 遗忘（删 / 并 / 降级提议） | 每周代理 | 只做减法 + 机制自检 |

---

## ★ 机制对自己的审查（它必须服从自己的规则）
把「默认特例、通用靠挣」**用在它自己身上**——这套机制本身，是不是「刚好解释了我们这次对话、就被当成通则」的伪通用？

诚实证伪——**它在哪失效？**
> 在**时间紧迫、必须当场就用一个尚未被验证的模型**的单次决策里，「靠 3 次独立使用慢慢挣」太慢，会**饿死决策**。

**结论 / 边界声明**：
> 本机制是「**长期养一座耐用知识库**」的**领域专用工具**，**不是认知的通用法则**。
> 已知失效边界 = 实时单次决策。

**紧急 override**：当你确需立刻把某模型当通用来用，可手动把其 `scope_type` 直接设为 `"通用"`，并在 `proven_domains` 注明「override + 理由」。机制不阻拦你，只要求你诚实留痕。

> 它用自己的规则，宣布了自己的边界——所以它没有犯它要防的那个错。
