# 思维模型库 · 收录与剪枝标准（防过拟合手册）

> 这套标准是整个系统的"正则项"。目标：让库**每天长大，但不长歪**——
> 不断扩展边界、补充缺失区间，同时拒绝冗余、狭窄、凑数的"伪模型"。
> **单一真相源 = `思维模型库.js`；本文件 = 维护它的规则。**

---

## 0. 核心心法：把"模型库"当成一个要泛化的模型来训练

你收集的不是知识点，而是一组**能迁移到新问题上的认知工具**。用机器学习的语言对照——你会立刻明白"不要过拟合"在这里到底指什么：

| 机器学习 | 这个思维模型库 |
|---|---|
| 训练样本 | 每个收录的模型 |
| **过拟合** | 收了一堆只在单一狭窄情境管用、或彼此高度冗余的"模型"——库看着很大，遇到**新问题**却调不出有用的工具 |
| 泛化能力 | 每个模型都能迁移到 **≥3 个不同领域** |
| **正则化** | 每周的**剪枝/合并**——惩罚复杂度，砍掉边际价值低的节点 |
| 偏差-方差权衡 | 广度（覆盖边界）vs 深度（别过度细分一个领域） |
| 验证集 | "这个模型帮我解决过一个我**没预先想到**的新问题吗？" |
| 特征共线性 | 模型之间不该高度相关（= 冗余，该合并） |
| 维度灾难 | 模型/维度堆太多 → 检索不动、根本用不起来 |

**一句话：宁要 100 个能扛事的模型，不要 500 个互相重叠的标签。**

---

## 1. 入库三道闸门（全过才进核心库）

新模型要进**核心库**（`status:"core"`），必须同时通过：

- **① 泛化闸**：能自然迁移到 **≥3 个不同领域**（`general ≥ 3`）。
  只在一个狭窄场景成立的"技巧/经验"→ 不是模型。
- **② 正交闸**：与现有模型**实质不同**。
  若它只是某个已有模型的**特例/换名**（如"晚买电脑后悔" ⊂ 机会成本）→ 当**子注**写进母模型，**不开新节点**。
- **③ 溯源闸**：有**真实的学术或思想出处**（论文 / 经典著作 / 公认提出者）。
  当周博主造的热词、无法溯源的"金句"→ 不收。

> 一票否决：**不可操作**（知道了也不改变任何决策/行动）的概念，再漂亮也不收。

### 入库前 5 问（30 秒自检）
1. 我能立刻说出它在 **3 个不同领域**的用法吗？（说不出 → 留待审或丢）
2. 库里**已有的哪个模型**和它最像？像到能合并吗？
3. 出处是谁、哪一年、哪本书/期刊？
4. 它会**改变我的某个具体决策**吗？怎么改？
5. 它落在**覆盖度地图哪一格**？是不是在补空格（而非往已满的格里再堆）？

---

## 2. 每日 SOP（≤5 分钟）

> 原则：**慢即是快**。每天最多沉淀 **1–2 个**，宁缺毋滥。冲动收录 = 对"今天读到的东西"过拟合。

1. **捕获**：遇到候选，先记一行，**别急着入库**（隔夜再看，热度退了还想留 = 真有用）。
2. **过闸门**：跑一遍上面三道闸 + 5 问。任一不过 → 丢弃，或先观察。
3. **定坐标**：选 1 个 `disc`（学科根源）+ 1 个 `func`（主要认知动作）。**优先填覆盖度地图里的暗红空格**。
4. **入待审**：在 `思维模型库.js` 末尾追加一行，`status:"candidate"`、`added` 写当天日期。
5. **升核心**：满足"能举 3 领域用例 + 隔几天仍觉得有用"→ 把 `status` 改成 `"core"`。

---

## 3. 每周"正则化"剪枝（10 分钟 · 防过拟合的核心动作）

光加不减，必然过拟合。每周固定做一次**减法**：

- [ ] **查失衡**：打开覆盖度地图——有没有某一类膨胀到占库 ⅓ 以上？过度生长的枝要停。
- [ ] **并近邻**：两两扫一遍，高度相似的合并，把弱的那个降为母模型的子注。
- [ ] **删死木**：一周内加了却**从没在真实决策里调用过、也举不出新用例**的 → 删或退回待审。
- [ ] **清待审**：待审区每个候选——要么升核心，要么删，**不许无限滞留**。

> 剪枝标准就是入库标准的逆运算：**违反三道闸门里任意一条的，就该被剪。**

---

## 4. 每月"扩展边界"复盘（看清盲区）

- 看覆盖度地图的**空行 / 空列** → 那是你的**认知盲区**（某个学科或某种认知功能你完全没有工具）。
- 必要时给 `disciplines` / `functions` **加新维度**来扩展边界，例如：
  - 学科：`生物与进化`、`伦理与价值`、`计算与信息`、`历史与制度`
  - 功能：`沟通/叙事`、`创造/生成`、`谈判/影响`、`时间/优先级`
- ⚠️ **目标是覆盖均匀，不是把每个格子填满**。强行凑满 64 格 = 另一种过拟合。空着也是一种诚实。

---

## 5. 🚨 过拟合红色警报（出现就停手剪枝）

- 一周加了 **>7 个**，且多数想不起在**第二个领域**怎么用。
- 新模型其实是旧模型**换了个名字**。
- 某一类**膨胀**到占全库三分之一以上。
- 你能背出定义，却**举不出 3 个不同领域**的用例。
- 加进来后，它**从没在一次真实决策里被你调用**过。
- 你在追**当周热点**的书 / 博主造的新词。
- 你开始收录"**为了完整**"而非"**为了有用**"的模型。

## 6. 字段填写规范

| 字段 | 怎么填 |
|---|---|
| `id` | 核心用 `m##`，待审用 `c##`，顺延不复用 |
| `disc` | 模型**思想根源**的学科，选 1 个（覆盖度地图的行） |
| `func` | 它**主要**帮你完成的认知动作，选 1 个（地图的列）；次要用途写进 `use` 文本即可 |
| `general` | 能自然迁移到几个不同领域，1–5；**<3 不进核心库** |
| `added` | 收录日期 `YYYY-MM-DD`，用于追踪生长速度 |
| `status` | `candidate`（待审外环）→ 审核通过 → `core`（核心库） |

## 7. 现成的「每日代理」提示词（可直接交给 /schedule 自动跑）

> 半自动模式：让代理每天**提议**，你**审核**。把下面整段作为定时代理的任务即可。

```
你是「思维模型库」的每日策展助手。数据源：C:\Users\29980\Desktop\思维模型\思维模型库.js

今天的任务（严守"宁缺毋滥、不要过拟合"）：
1. 先读取 思维模型库.js，了解已有模型，以及覆盖度地图里的空格（disc×func 为 0 的组合）。
2. 从权威学术文献（顶级期刊 / 经典著作 / 公认提出者）中，物色 **最多 1 个** 新候选模型，
   优先填补"覆盖度空格"或欠缺的学科/功能维度——即"扩展边界、补充缺失区间"。
3. 让它通过三道闸门，否则**今天就不加**（允许空手而归，这比凑数好）：
   ① 泛化：能迁移到 ≥3 个不同领域（general≥3）；
   ② 正交：与库中现有模型实质不同，不是特例/换名；
   ③ 溯源：有可靠学术/思想出处。
4. 产出一个待审条目（status:"candidate"），格式与 思维模型库.js 中现有对象完全一致，
   字段齐全：id(c##顺延)/cat/cn/en/principle/scope/use/src/disc/func/general/added(今天)/status。
5. 附上 3 句话：①它属于哪个覆盖度空格、为何值得补；②它能迁移的 3 个领域各举一例（证明不过拟合）；
   ③库里和它最接近的现有模型是哪个、差异在哪（证明正交）。
6. 把候选**追加**进 思维模型库.js 的待审区（绝不直接动核心库、不改已有条目），等用户审核升核心。
7. 每周一额外做一次"正则化"：列出建议合并/删除的冗余或低泛化模型，但只提议、不擅自删。
```

## 8. 操作速查

| 想做的事 | 怎么做 |
|---|---|
| **加一个模型** | 在 `思维模型库.js` 的 `models` 末尾追加一行对象，存盘，刷新 HTML |
| **升级待审 → 核心** | 把该条 `status:"candidate"` 改成 `"core"` |
| **剪掉一个模型** | 删除该行（或先退回 `candidate` 观察） |
| **扩展新维度** | 往 `disciplines` 或 `functions` 数组加一个新字符串 |
| **看缺口** | 打开 HTML → 顶部切到「覆盖度地图」→ 看暗红空格和"补充建议" |
| **改更新日期** | 改顶部 `updated` 字段（新加模型会高亮"NEW"） |

> 维护好这一个 `.js` 文件，HTML 永远自动跟着更新。**这就是"可迭代"。**
